Описание того, какой логикой автор руководствовался в ходе принятия решений и выбора технологий. Учтены все функциональные и нефункциональные требования.

Принципиально, чтобы экспорт файла ставок не был завязан на сохранение ставки в API и не блокировал процесс сохранения.
Таким образом это осуществляется через Kafka. API создает сообщение об изменении ставки и специальный Rate Consumer экспортирует файлы.
Так же это можно сделать с помощью Kafka Connect SFTP Sink. 

### <a name="_bjrr7veeh80c"></a>**Альтернативы**
Описание наиболее важных альтернативных решений.

1. Можно было экспортировать ставки не по факту их изменения, а периодически.
Тогда в актуальности информации по ставкам образуется разрыв.

2. Можно экспортировать ставки в API Deposit Service синхронно в методе сохранения ставок,
но это увеличит отклик метода и само сохранение может заканчиваться неудачей только из-за недоступности sftp-диска к примеру.
